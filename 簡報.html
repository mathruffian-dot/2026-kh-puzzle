什麼是 Gem？

Gem = 可自訂的專屬 AI 助手

  • 一次設定出題規則與方向
  • 每次只需輸入謎底
  • 自動產出風格一致、邏輯穩定的題目
  • 減少重複出題與調整的負擔

運作流程

老師輸入：MATH ↓ Gem 自動處理： M → 鋁 (Al) A → 氫 (H) T → 鈣 (Ca) H → 氧 (O) ↓ 產出兩張圖： 圖 A：謎題圖（中文名） 圖 B：對照表圖

Gem 的七段結構

1. profile（身分資料）

name / role / description
= Gem 的名牌

2. knowledge_base（知識庫）

翻譯規則 = Gem 的靈魂
直接寫出完整對應表

3. system_behavior（系統行為）

指令 + 工作流程 + 禁令
= Gem 的做事 SOP

4. puzzle_image（謎題圖設計）

雙模式「視覺風格」：
沉浸（古卷軸）+ 教學（清爽）

5. reference_chart（對照表圖）

學生解謎用的道具圖
印 A4 發下去

6. user_manual（使用手冊）

老師用：推薦謎底＋風險提醒

💡 第 4 點的「雙模式」是圖的視覺風格，不是劇情——劇情是 Stage 2 劇本 Gem 負責

兩種設計模式

模式 A：靜默生圖

老師輸入 → AI 直接產圖

✓ 快速批量產題

✓ 對應簡單（A-Z 摩斯、A1Z26）

✓ 謎底短（2-4 字）

代表：摩斯、等差、代數偏移鎖、方程式圖形

模式 B：先驗算後生圖

老師輸入 → AI 驗算表 → 確認 → 產圖

✓ AI 易出錯的知識（中文筆畫、20+ 元素）

✓ 精確度要求高（時鐘、座標）

✓ 正式使用（研習、比賽）

代表：元素週期表、時鐘、筆畫、注音、音節

範例 3：等差數列密碼

輸入：4826 ↓ 古卷軸顯示四排數列： 2, 5, 8, 4 , 11, 14, 17   ← 公差 3，入侵者是 4 9, 7, 5, 3, 8 , -1, -3   ← 公差 -2，入侵者是 8 2 , -5, -10, -15, -20, -25, -30   ← 公差 -5，入侵者是 2 4, 10, 16, 22, 6 , 34, 40   ← 公差 6，入侵者是 6 ↓ 答案：4826

設計巧思：公差不能是 ±1（避免學生猜）｜支援遞增/遞減｜可加風格關鍵字（如「3829 賽博龐克」）

範例 4：二元一次方程式圖形

數字 → 方程式映射

0: y = k 水平線（上） 1: y = -k 水平線（下） 2: x = k 鉛直線（右） 3: x = -k 鉛直線（左） 4: x - y = -k ↗ 避開 Q4 5: x - y = k ↗ 避開 Q2 6: x + y = k ↘ 避開 Q3 7: x + y = -k ↘ 避開 Q1 8: x - y = 0 過原點 ↗ 9: x + y = 0 過原點 ↘

設計亮點：兩層反查（元素週期表）

謎題圖只顯示： 鋁、氫、鈣、氧 學生第一層查： 鋁 → Al、氫 → H、鈣 → Ca、氧 → O 學生第二層查： Al → 13 → M、H → 1 → A、Ca → 20 → T、O → 8 → H → 得出謎底：MATH

設計巧思：用中文名而非符號，迫使學生必須熟悉「中文名 ↔ 元素符號」對應——不能靠背符號速通，真正測驗週期表學習成效。

Stage 2：情境解謎敘事建築師

輸入（四欄 schema）

theme: 兒童節守門魔神 style_preset: 奇幻 puzzles: - {level: 1, password: "CHILD", puzzle_type: 摩斯, context: "泛黃電碼對照表"} - {level: 2, password: "TEENAGER", puzzle_type: 代數偏移鎖, context: "門上方程式＋亂碼"} ...

Gem 強制 4 個原則

  • 難度曲線：前易後難、情感漸強
  • 每關動機：主角內心為何要解這題
  • 關卡橋接：前關道具成為後關線索
  • 結局回呼：結尾必須呼應開場意象

輸出含 7 段落 + 技術參考 JSON（給 Stage 3 吃）

Stage 3：解謎遊戲網頁建構師

輸出：單一 self-contained HTML｜iPad 橫向優化

N 個點，解鎖變綠

🎵 音效

解鎖音、失敗音、BGM（使用者互動後啟動）

🎨 圖片怎麼換？用氛圍編程（Vibe Coding）

不用改程式碼。直接跟 AI 說：「幫我把第一關謎題圖換成 https://duk.tw/xxx.jpg」或「把對照表替換成我剛上傳的 morse-chart」，AI 會自動找到對應位置並更新。按完按鈕後，就會跳出你指定的圖片。

完整實戰流程

Step 1 用 Stage 0 產生器 設計自己的 Stage 1 Gem（只做一次） Step 2 用 Stage 1 Gem 產各關謎題圖與對照表（每次活動 N 關） Step 3 上傳 duk.tw 得到 8 張圖的直連 URL Step 4 用 Stage 2 Gem 輸入謎題清單，得劇本＋technical_ref Step 5 用 Stage 3 Gem 貼劇本，得單一 .html 檔 Step 6 替換圖片 搜尋 data-placeholder 替換 src Step 7 發布 GitHub Pages / Netlify Drop / Canvas 共用
